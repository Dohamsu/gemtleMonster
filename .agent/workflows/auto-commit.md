---
description: 변경 사항을 스테이징하고, 내용을 분석하여 한국어 커밋 메시지로 커밋한 후 푸시합니다.
---

1. `git add .` 명령어를 실행하여 모든 변경 사항을 스테이징합니다.
2. `git status` 또는 `git diff --staged`를 사용하여 변경된 내용을 분석합니다.
3. 변경 내용을 바탕으로 명확하고 간결한 한국어 커밋 메시지를 작성합니다.
   - 새로운 기능 추가: "기능 추가: [기능 설명]"
   - 버그 수정: "버그 수정: [수정 내용]"
   - 리팩토링: "리팩토링: [내용]"
   - 기타: 적절한 접두사를 사용하거나 내용을 서술합니다.
4. 작성한 메시지로 커밋합니다: `git commit -m "[작성한 메시지]"`
5. 원격 저장소로 푸시합니다: `git push`
