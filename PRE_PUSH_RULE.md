# 🚀 Git 푸시 전 사전 검증 규칙 (Pre-Push Verification)

이 규칙은 깃(Git)에 코드를 푸시하기 전에 코드의 안정성과 품질을 보장하기 위해 반드시 수행해야 하는 단계들을 정의합니다.

## 📋 사전 필수 체크리스트

### 1. 런타임 오류 확인 (Runtime)
- **방법**: `npm run dev`를 실행하고 브라우저를 엽니다.
- **확인 사항**:
    - 브라우저 콘솔(`F12`)에 빨간색 에러(Error)가 없는가?
    - Supabase API 호출 시 404 또는 406 오류가 발생하는 상황이 완벽히 제어되었는가? (예: `.maybeSingle()` 사용)

### 2. 린트 오류 수정 (Lint)
- **방법**: 터미널에서 `npm run lint`를 실행합니다.
- **확인 사항**:
    - "0 errors" 결과가 나오는가?
    - 사용하지 않는 변수(unused variables)나 타입 불일치 이슈가 모두 해결되었는가?

### 3. 빌드 성공 여부 확인 (Build)
- **방법**: 터미널에서 `npm run build`를 실행합니다.
- **확인 사항**:
    - TypeScript 타입 체크(`tsc`)를 통과했는가?
    - Vite 빌드 프로세스가 에러 없이 완료되었는가?

### 4. 테스트 코드 검증 (Test)
- **방법**: 터미널에서 `npm test`를 실행합니다.
- **확인 사항**:
    - `src/lib/battleUtils.test.ts` 등 주요 로직의 테스트 케이스가 통과하는가?

---

> [!IMPORTANT]
> **위 세 가지(런타임, 린트, 빌드) 중 하나라도 실패하면 깃에 푸시하지 않습니다.** 오류를 먼저 수정한 후 다시 검증을 거쳐 푸시를 진행하십시오.
